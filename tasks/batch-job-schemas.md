  ---
  Pydantic Schemas (v2/schemas.py)

  BatchItemStatus (returned inside status response)
  id: int
  position: int
  filename: str
  candidate_name: str
  candidate_university: str | None
  status: str                    # "pending" | "running" | "completed" | "error"
  error_message: str | None
  repo_id: int | None            # only set when completed
  verdict: Verdict | None        # only set when completed
  standout_features: list[str]   # populated when completed, may be empty list

  BatchStatusResponse (returned by GET /batch/{batch_id}/status)
  batch_id: str
  created_at: datetime
  total_items: int
  completed_items: int
  status: str                    # "pending" | "running" | "completed"
  items: list[BatchItemStatus]

  BatchUploadResponse (returned by POST /batch/upload)
  batch_id: str

  ---
  Acceptance Criteria

  - BatchJob and BatchItem tables are created on app startup via Base.metadata.create_all()
  - All Pydantic schemas validate correctly
  - Foreign key cascade: deleting a BatchJob deletes all associated BatchItem rows
